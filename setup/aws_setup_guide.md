# AWS Infrastructure Setup for Attendify

This guide provides step-by-step instructions for setting up the necessary AWS infrastructure for the Attendify Capstone project, including account creation, IAM configurations, Lambda deployments, and configuring your local environment variables.

---

## 1. Create an AWS Account

If you don't already have an AWS account, you'll need one to host the backend services.

1. Go to the [AWS Free Tier Sign Up Page](https://aws.amazon.com/free/).
2. Click **Create a Free Account** (or "Complete Sign Up").
3. Enter your **Email Address** and a desired **Account Name**. Verify your email using the verification code sent to you.
4. Set up a strong, secure password for your AWS root user.
5. Provide your contact information (Personal or Business).
6. Enter your billing information (AWS requires a credit/debit card for verification but won't charge you if you stay within the Free Tier limits).
7. Complete identity verification (normally via SMS).
8. Select the **Basic Support - Free** plan.
9. Wait a few moments for AWS to activate your account. You will receive an email once it is ready.

---

## 2. Create an IAM User

It is a security best practice **not** to use your root account for daily operations or API access. We will create an IAM (Identity and Access Management) user specifically for Attendify.

1. Log in to the [AWS Management Console](https://console.aws.amazon.com/) using your root account.
2. In the search bar at the top, type `IAM` and select the **IAM** service.
3. In the left navigation pane, click on **Users**, then click the **Create user** button.
4. **User details:**
   - **User name:** Enter `attendify-admin` (or another descriptive name).
   - Check the box **"Provide user access to the AWS Management Console"**.
   - Select **"I want to create an IAM user"**.
   - **Console password:** Choose "Autogenerated password" or "Custom password" as you prefer.
   - Click **Next**.

## 3. Assign Required Permissions

In the "Set permissions" step for your new IAM User, do the following:

1. Select **Attach policies directly**.
2. Search for and check the following managed policies, which are required for the Attendify system to function:
   - `AmazonDynamoDBFullAccess` (Required for the Attendance and Employee tables)
   - `AmazonRekognitionFullAccess` (Required for indexing and searching faces via Image/Collection APIs)
   - `AWSLambda_FullAccess` (Required for creating and modifying the serverless functions)
   - `AmazonAPIGatewayAdministrator` (Required to connect your Lambdas to HTTP endpoints for the Flutter apps to call)
   - `CloudWatchLogsFullAccess` (Required for debugging Lambda functions and monitoring system health)
3. Click **Next** to review, and then click **Create user**.
4. **IMPORTANT CRITICAL STEP:** On the success screen, click **Download .csv file** or copy the **Console sign-in URL**, Username, and Password. *You will not be able to retrieve the password again.*
5. **Sign out** of your Root account, and **Sign in** using the new IAM User credentials and Console sign-in URL you just saved.

---

## 4. Setting up serverless Lambda Functions

Attendify utilizes AWS Lambda functions as its backend API. You will need to recreate these functions in your AWS console and paste the appropriate code into them.

All Lambda function code for this project is meant to be placed into the local `lambda/` directory within this repository so you can easily manage it.

### Step-by-Step Instructions for Creating a Lambda Function:
1. Log in to the AWS Console with your `attendify-admin` IAM user.
2. In the search bar, type `Lambda` and click the **Lambda** service.
3. Make sure you are in the correct Region (e.g., `ap-south-1` for Mumbai) using the dropdown in the top right.
4. Click **Create function**.
5. Choose **Author from scratch**.
6. **Function details:**
   - **Function name:** You must create functions matching these exact names:
     - `get_all_employeesLambda`
     - `user-service`
     - `manual_attendanceLambda`
     - `attendance_statsLambda`
     - `delete_userLambda`
     - `recent_attendanceLambda`
     - `mark_attendanceLambda`
   - **Runtime:** Select `Python 3.14` (or whatever your AWS region supports natively).
   - **Architecture:** Keep it as `x86_64` (default).
7. Under **Permissions**, let it create a new role with basic Lambda permissions. *(You will need to extend this role's permissions to allow the Lambda to access DynamoDB and Rekognition from the IAM Console).*
8. Click **Create function**.

### Step-by-Step Instructions to Deploy Your Code:
1. After the function is created, scroll down to the **Code source** section.
2. Open the corresponding `.py` file stored locally in the `lambda/` folder of this repository.
3. **Copy all the code** from your local file. *Note: I have given you the Lambda function code in the `lambda/` folder, where all the code for each function is given. Ensure you give the name of the Lambda function in AWS exactly as per the file name.*
4. **Paste the code** into the online AWS editor, overwriting the sample code.
5. Click the gray **Deploy** button above the editor to save.
6. Repeat this process for every function needed.

## 5. Configuring Local `.env` Secrets File

To connect your Flutter applications (Kiosk and Dashboard) to your new AWS backend, you need to create a file named `.env` in the root of the respective app directory (e.g. `kiosk_app/.env` and `dashboard_app/.env`).

Below is how to find and configure every necessary value located in AWS:

### 1. API_BASE_URL
This is your API Gateway endpoint that connects your application to the Lambda functions.
1. Search and open the **API Gateway** service in AWS.
2. Select your created API (e.g., "Attendify API").
3. In the left hand sidebar, click on **Stages**.
4. Click on your active deployment stage (usually `default` or `prod`).
5. At the top of the panel, you will see an **Invoke URL**. 
6. Copy that URL and paste it into your `.env` file like this:
   `API_BASE_URL=https://<your-api-id>.execute-api.<region>.amazonaws.com/default`

### 2. AWS_REGION
This represents the geographic region hosting your backend infrastructure.
1. Look at the top-right corner of the AWS Management Console navigation bar.
2. You will see a dropdown with a city name (e.g., "Mumbai"). Click the dropdown to see the region code next to it (e.g., `ap-south-1`).
3. Copy that code and paste it into your `.env` like this:
   `AWS_REGION=ap-south-1`

### 3. COGNITO_USER_POOL_ID
This is the identifier for the directory of users managing dashboard authentication.
1. Search and open **Amazon Cognito** in AWS.
2. Click on **User pools** and select your created User Pool.
3. On the **User pool overview** page, copy the **User pool ID** (it usually looks like `ap-south-1_xxxxxxxxx`).
4. Paste it into your `.env` file like this:
   `COGNITO_USER_POOL_ID=ap-south-1_LFND...`

### 4. COGNITO_CLIENT_ID
This is the specific app client configuration ID verifying your Flutter app's right to use the Cognito User Pool.
1. While still viewing your User Pool in Amazon Cognito, click on the **App integration** tab.
2. Scroll to the bottom to find the **App client list**.
3. Copy the **Client ID** (a long alphanumeric string) for your specific app client.
4. Paste it into your `.env` file like this:
   `COGNITO_CLIENT_ID=kfj31aolej...`

### 5. S3_BUCKET_NAME
This represents the cloud storage bucket containing employee facial profile images.
1. Search and open the **S3** service in AWS.
2. Click on **Buckets** from the dashboard.
3. Find the bucket you provisioned for the images (e.g., `faceattend-images-sakshi`).
4. Copy the exact text of the bucket's name securely.
5. Paste it into your `.env` file like this:
   `S3_BUCKET_NAME=faceattend-images-sakshi`

### Final Result
Your resulting `.env` file should look similar to this:
```env
API_BASE_URL=https://s3c1f3w0jg.execute-api.ap-south-1.amazonaws.com/default
AWS_REGION=ap-south-1
COGNITO_USER_POOL_ID=ap-south-1_LFNDMTkIu
COGNITO_CLIENT_ID=kfj31aolej424vk3fv7rj02rk
S3_BUCKET_NAME=faceattend-images-sakshi
```
